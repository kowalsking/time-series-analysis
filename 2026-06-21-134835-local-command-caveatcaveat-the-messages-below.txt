     PRD: Аналіз часових рядів повітряних тривог в Україні

     Context (Навіщо)

     Повітряні тривоги — критичний сигнал безпеки для мільйонів людей. Історичні
     дані
     про тривоги дозволяють виявляти патерни (коли найчастіше оголошують
     тривоги,
     у яких регіонах, як довго вони тривають) і будувати короткострокові 
     прогнози
     навантаження. Це навчальний MVP, що демонструє повний цикл аналізу часових
     рядів:
     від сирих даних → до інтерактивного дашборда з прогнозом.

     Проблема: немає простого інструменту, щоб подивитися історичні патерни
     тривог і
     оцінити, наскільки вони прогнозовані.
     Очікуваний результат: Streamlit-дашборд, який показує статистику, патерни
     за
     годинами/днями/регіонами та короткостроковий прогноз кількості тривог.

     Зафіксовані рішення

     - Дані: Kaggle dimakyn/air-alarm-ukrain-2022022420220409 (історичні події
     тривог по регіонах, ~24.02.2022–09.04.2022). Завантажується вручну в
     data/raw/.
     - Аналітична мета: візуалізація/дашборд + короткостроковий прогноз.
     - Формат: веб-дашборд на Streamlit.
     - Рівень: навчальний MVP — простий, читабельний код, базові-середні методи.
     - Часовий пояс: Europe/Kyiv для всіх часових агрегацій.

     ▎ ⚠️ Точну схему CSV не вдалося підтвердити автоматично (Kaggle рендериться
     ▎ через JS).
     ▎ Перший крок реалізації — інспекція реальних колонок 
     ▎ (notebooks/00_inspect.ipynb
     ▎ або короткий скрипт), після чого зафіксувати маппінг у src/config.py.
     ▎ Очікувані поля: регіон/область, час початку, час завершення тривоги.

     Технологічний стек

     - Python 3.11+, pandas, numpy — обробка даних
     - streamlit — дашборд
     - plotly — інтерактивні графіки (вбудовуються в Streamlit)
     - statsmodels — сезонна декомпозиція + прогноз (Holt-Winters / SARIMAX)
     - pytest — тести препроцесингу
     - (опц.) kaggle CLI — завантаження датасету

     Конвеєр обробки даних

     1. Завантаження — користувач кладе CSV у data/raw/ (інструкція в README).
     2. data_loader.load_raw() — читає CSV, повертає сирий DataFrame.
     3. preprocess.build_events() — канонічна таблиця подій:
     [region, start_dt (tz=Europe/Kyiv), end_dt, duration_min].
     Парсинг дат, нормалізація назв регіонів, відсів некоректних/негативних
     тривалостей.
     4. features.py — будує часові ряди:
       - hourly_active_regions() — к-сть регіонів під тривогою за кожну годину;
       - hourly_alert_starts() — к-сть початків тривог за годину (цільовий ряд
     прогнозу);
       - per_region_series(), durations().
     5. Збереження очищених даних у data/processed/ (кеш для дашборда).

     Аналітичні модулі (analysis.py)

     - KPI: всього тривог, сумарні години тривог, середня тривалість, к-сть
     регіонів.
     - Розподіл за годиною доби (24 біни) і за днем тижня.
     - Денний тренд (к-сть тривог по днях).
     - Топ регіонів за к-стю/сумарною тривалістю.
     - Heatmap регіон × година доби.
     - Кореляція між регіонами (індикатор масованих, синхронних тривог).

     Прогнозування (forecast.py)

     - Цільовий ряд: погодинна к-сть початків тривог (національний рівень).
     - Train/test split: останні ~7 днів — тест.
     - Моделі:
       a. Наївний сезонний baseline — «та сама година попереднього дня» (період
     = 24h).
       b. Holt-Winters (statsmodels ExponentialSmoothing, seasonal_periods=24).
       c. (опц.) SARIMAX з добовою сезонністю для порівняння.
     - Метрики: MAE, RMSE (порівняння моделей із baseline).
     - Прогноз на наступні 24–48 год із довірчим інтервалом.

     ▎ Примітка: датасет короткий (~45 днів), тому тижневу/довгу сезонність не 
     ▎ показуємо —
     ▎ фокус на добовому циклі та короткостроковому горизонті. Це чесно 
     ▎ зазначаємо в дашборді.

     Дашборд (app/dashboard.py)

     Streamlit, @st.cache_data для завантаження/обробки. Сайдбар: вибір
     регіону(ів),
     діапазон дат. Вкладки:

     1. Огляд — KPI-карти + загальний часовий ряд активних тривог.
     2. Патерни — графік за годиною доби, за днем тижня, heatmap регіон×година,
     топ регіонів.
     3. Прогноз — фактичний ряд + прогноз на 24–48 год, таблиця метрик моделей.

     Усі графіки будуються через функції з src/viz.py (немає дублювання логіки).

     Етапи реалізації (порядок)

     1. Скелет: requirements.txt, .gitignore, README.md, пакет src/, config.py.
     2. Інспекція реальної схеми CSV → зафіксувати маппінг колонок у config.py.
     3. data_loader + preprocess (+ tests/test_preprocess.py).
     4. features — часові ряди.
     5. analysis — патерни та статистика.
     6. forecast — моделі + метрики.
     7. viz — графіки plotly.
     8. app/dashboard.py — збірка дашборда.
     9. README: інструкції завантаження даних і запуску.

     Верифікація

     - Дані: python -c "from src.preprocess import build_events; 
     print(build_events().head())"
     — таблиця подій коректна, дати в tz Europe/Kyiv, тривалості додатні.
     - Тести: pytest tests/ — проходять (парсинг дат, нормалізація регіонів,
     тривалості).
     - Прогноз: python -c "from src.forecast import evaluate; print(evaluate())"
     — Holt-Winters має MAE/RMSE не гірше за наївний baseline.
     - Дашборд: streamlit run app/dashboard.py — усі 3 вкладки рендеряться,
     фільтри за регіоном/датою працюють, графіки оновлюються, прогноз
     відображається.
     - End-to-end: свіже клонування → встановити залежності → покласти CSV →
     запустити
     дашборд за README без помилок.
